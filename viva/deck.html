§ 03 — Why this matters

A single modality is an under-determined signal.

Fake profiles power romance scams, financial fraud, election interference and harassment campaigns. Detection systems today look at one modality at a time — and an attacker only needs to beat that one.

Attack surface today
  • GAN / diffusion faces defeat image-only filters
  • LLM-paraphrased bios defeat text-only filters
  • Stripped EXIF defeats metadata-only filters
  • No system explains why a profile was flagged
Our hypothesis
  • Faking all three modalities consistently is hard
  • Ensembles + weighted fusion outperform any single model
  • Explanations (Grad-CAM, SHAP/LIME) are the actual product
  • A calibrated trust score > a binary "fake / real" label

§ 06 — Proposed solution

Three lanes, one verdict.

A profile is dispatched to three independent analysis modules, each an ensemble. Module outputs (score, confidence, evidence) are aggregated by a weighted fusion engine into a single calibrated trust score T ∈ [0,1].

  1. Profile in
    User uploads a profile image (≤10 MiB) and a bio text (10–1000 chars) through the React frontend or REST API.
  2. Run three pipelines in parallel
    Image ensemble + Grad-CAM · Text ensemble + LIME · Metadata forensics + EXIF anomaly scoring.
  3. Fuse with adaptive weights
    If a modality is missing or low-confidence, weights redistribute automatically — the system degrades but it never lies.
  4. Return a verdict + an explanation
    Trust score ∈ [0,1], trust level, contributing factors, Grad-CAM overlay, token highlights.
Design principles
A.
Ensemble > single model. Empirical weights, not arbitrary.
B.
Explainability is the product. A score without an explanation is a guess.
C.
Calibrated uncertainty. Low-confidence predictions are pulled toward 0.5.
D.
Local-first. All inference runs on-device. No third-party cloud calls.

§ 08 — Module 01 / Image

Visual ensemble · spatial attention.

The image is normalised and resized for each backbone, then routed through three classifiers whose softmax probabilities are blended with empirical weights. The dominant signal is EfficientNet-B7 (50%), backed by XceptionNet (40%) — the gold-standard deepfake detector — and CLIP ViT-B/32 (10%) for general vision-language sanity checking. YOLOv8 face-cropping runs when a face is detected and falls back to whole-image analysis otherwise.

§ 09 — Module 02 / Text

Text ensemble · token attribution.

Bios are tokenised and scored by an ensemble of two transformer classifiers and a rule-based linguistic detector. The dominant signal is the OpenAI RoBERTa AI-text detector (70%), supported by the ChatGPT-detector (20%) for GPT-family outputs, with a rule-based heuristic (10%) catching obvious patterns the transformers miss.

§ 10 — Module 03 / Metadata

EXIF forensics · anomaly scoring.

Generated images rarely carry a consistent EXIF chain. The metadata module reads every available EXIF field and, crucially, examines what is missing. It scores four independent forensic signals and fuses them into a value in [0, 1]. When an uploaded image has its EXIF stripped, which is common on social platforms, the adaptive weights on slide 12 take over and the fusion proceeds on image and text alone.

§ 12 — Trust Score Computation

Algorithm 1 · compute_trust_score.

Algorithm 1 · compute_trust_score src/modules/fusion/trust_scorer.py
input  rI, rT, rM  # ModuleResult | None
output  TrustScoreResult  # score, level, interpretation
const  w = { I: 0.4,  T: 0.3,  M: 0.3 }
const  θ = { vlow: 0.30,  low: 0.50,  mod: 0.70 }
S, C ← {}, {}
for module m in { I, T, M } if rm ≠ None do
S[m] ← rm.score    C[m] ← rm.confidence
if S = ∅ then raise ValueError
# Eq. 2 · renormalise weights over available modules
Z ← Σm∈S w[m]
w'[m] ← w[m] / Z    ∀ m ∈ S
# Eq. 1 · weighted aggregation
T ← Σm∈S w'[m] · S[m]
level ← band(T, θ)
σ ← std(S.values())  # module disagreement
c̄ ← mean(C.values())  # avg. confidence
note ← interpret(T, level, σ, c̄)
return  { T, level, note, S, C, w' }
Invariants
  • T ∈ [0, 1]   — each sm ∈ [0,1], Σw' = 1
  • Σ w'[m] ≡ 1   — Eq. 2 guarantees normalisation
  • Output is deterministic given module results
  • Module failure ≠ pipeline failure (graceful degrade)
Complexity
Fusion: O(k)   k = |modules| = 3
Image: O(W·H)   CNN forward pass
Text: O(L2)   transformer attention
LIME: O(n·L)   n=50 samples
Interpretation function
band(T, θ):
T < 0.30 → very_low
T < 0.50 → low
T < 0.70 → moderate
else    → high
σ < 0.1 → "strongly agree"; < 0.2 → "generally agree"; else → "conflicting".

Conclusion

A multimodal, explainable, locally-runnable fake-profile detector — honest about what it can and can't see today and architected to keep improving.
